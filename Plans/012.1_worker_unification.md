# Worker Unification — Engine Internalization

**Status:** In Progress
**Started:** 2026-03-25
**Parent Phase:** 12.1 from Master Plan

---

## Goal

Make IRT's backtest worker fully self-contained with zero dependency on the Operations-Platform repository. Copy the backtest engine and ibkr-core packages locally, create a project venv with all dependencies, and point the worker to the local engine.

## Background

Phase 12.1 was originally scoped as "port parallel orchestrator from Ops Platform". That part is done (worker/orchestrator.py, main.py, config.py). However, the engine itself was still invoked as a subprocess pointing to `Operations-Platform/packages/backtest-engine/main.py`, causing `ModuleNotFoundError: ibkr_core` when running the worker standalone.

The redesigned scope internalizes everything so IRT is independent.
