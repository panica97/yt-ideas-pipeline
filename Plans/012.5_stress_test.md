## Goal

Implement a parameter sensitivity test that sweeps strategy parameters (indicator periods, thresholds, etc.) and runs backtests for each variation. Produces a robustness score based on percentage of profitable variations, positive Sharpe ratio, and low drawdown. Adapted from the Operations Platform's stress test, restructured to match IRT's modular architecture.

## Notes

- **Param override format**: `{"ind_list.1 day.0.period": {"min": 5, "max": 30, "step": 5}}` — dot-path notation for nested strategy JSON fields
- **Two sweep modes**: multi-param grid (cartesian product) and single-param sweeps (one-at-a-time)
- **Robustness score**: average of (profitable_pct + positive_sharpe_pct + low_drawdown_pct) / 3
- **Future enhancement**: visual param builder in frontend instead of raw JSON textareas
- **Source reference**: Ops Platform `StressTestArea.vue` (~1500 lines) has the full implementation in Vue/ECharts. We adapt to React/Recharts/Tailwind.
- **Future enhancement**: heatmap visualization for 2-param grids
